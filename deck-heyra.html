Setup

The entire integration. Seven lines of JSON.

.mcp.json
{
  "mcpServers": {
    "img": {
      "type": "http",
      "url":  "https://heyraimg-backend-production.up.railway.app/mcp"
    }
  }
}

Drop this file in your project root. Your AI agent picks it up automatically. Claude Code, Cursor, any MCP client.

What it solves

Questions you already have. Answered.

Where do the images come from?
The Heyra collection. 1,489 royalty-free photos, hosted on Cloudflare R2. No licensing issues.
How does the AI find the right image?
CLIP-based semantic search. The agent describes what the slide needs. Heyra returns ranked matches with relevance scores.
Does it work in Cursor?
Any MCP-compatible client. Claude Code, Cursor, or any tool that reads .mcp.json.
Can I add my own images?
Yes. The /api/index endpoint lets you index new folders. Your library grows with your content.

Why this matters

AI agents can write slide content. They can pick the right component. They can follow a storytelling framework. The missing piece was always the visuals. Heyra closes that gap. One MCP server gives every agent access to a curated image library with semantic search. The deck goes from outline to finished, visuals included, in a single conversation.
